  7  """ 
  8  Parallel acces to netCDF files 
  9   
 10  One netCDF dimension is defined for splitting the data among 
 11  processors such that each processor is responsible for one slice of 
 12  the file along that dimension. 
 13   
 14  Since netCDF files can be very big, the distribution algorithm gives 
 15  priority to memory efficiency over CPU time efficiency. The processor 
 16  that handles the file treats only one slice per superstep, which means 
 17  that at no time more than one slice must be stored in any processor.
