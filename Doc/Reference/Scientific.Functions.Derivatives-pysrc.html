  7  """ 
  8  Automatic differentiation for functions of 
  9  any number of variables up to any order 
 10   
 11  An instance of the class DerivVar represents the value of a function 
 12  and the values of its partial X{derivatives} with respect to a list of 
 13  variables. All common mathematical operations and functions are 
 14  available for these numbers. There is no restriction on the type of 
 15  the numbers fed into the code; it works for real and complex numbers 
 16  as well as for any Python type that implements the necessary 
 17  operations. 
 18   
 19  If only first-order derivatives are required, the module 
 20  FirstDerivatives should be used. It is compatible to this 
 21  one, but significantly faster. 
 22   
 23  Example:: 
 24   
 25    print sin(DerivVar(2)) 
 26   
 27  produces the output:: 
 28   
 29    (0.909297426826, [-0.416146836547]) 
 30   
 31  The first number is the value of sin(2); the number in the following 
 32  list is the value of the derivative of sin(x) at x=2, i.e. cos(2). 
 33   
 34  When there is more than one variable, DerivVar must be called with 
 35  an integer second argument that specifies the number of the variable. 
 36   
 37  Example:: 
 38   
 39      >>>x = DerivVar(7., 0) 
 40      >>>y = DerivVar(42., 1) 
 41      >>>z = DerivVar(pi, 2) 
 42      >>>print (sqrt(pow(x,2)+pow(y,2)+pow(z,2))) 
 43   
 44      produces the output 
 45   
 46      >>>(42.6950770511, [0.163953328662, 0.98371997197, 0.0735820818365]) 
 47   
 48  The numbers in the list are the partial derivatives with respect 
 49  to x, y, and z, respectively. 
 50   
 51  Higher-order derivatives are requested with an optional third 
 52  argument to DerivVar. 
 53   
 54  Example:: 
 55   
 56      >>>x = DerivVar(3., 0, 3) 
 57      >>>y = DerivVar(5., 1, 3) 
 58      >>>print sqrt(x*y) 
 59   
 60      produces the output 
 61   
 62      >>>(3.87298334621, 
 63      >>>    [0.645497224368, 0.387298334621], 
 64      >>>      [[-0.107582870728, 0.0645497224368], 
 65      >>>        [0.0645497224368, -0.0387298334621]], 
 66      >>>          [[[0.053791435364, -0.0107582870728], 
 67      >>>            [-0.0107582870728, -0.00645497224368]], 
 68      >>>           [[-0.0107582870728, -0.00645497224368], 
 69      >>>            [-0.00645497224368, 0.0116189500386]]]) 
 70   
 71  The individual orders can be extracted by indexing:: 
 72   
 73      >>>print sqrt(x*y)[0] 
 74      >>>3.87298334621 
 75      >>>print sqrt(x*y)[1] 
 76      >>>[0.645497224368, 0.387298334621] 
 77   
 78  An n-th order derivative is represented by a nested list of 
 79  depth n. 
 80   
 81  When variables with different differentiation orders are mixed, 
 82  the result has the lower one of the two orders. An exception are 
 83  zeroth-order variables, which are treated as constants. 
 84   
 85  Caution: Higher-order derivatives are implemented by recursively 
 86  using DerivVars to represent derivatives. This makes the code 
 87  very slow for high orders. 
 88   
 89  Note: It doesn't make sense to use multiple DerivVar objects with 
 90  different values for the same variable index in one calculation, but 
 91  there is no check for this. I.e.:: 
 92   
 93      >>>print DerivVar(3, 0)+DerivVar(5, 0) 
 94   
 95      produces 
 96   
 97      >>>(8, [2]) 
 98   
 99  but this result is meaningless.
